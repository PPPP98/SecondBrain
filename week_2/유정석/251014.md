## 아이디어 구체화

### Torch

#### 서비스 컨셉

- **웹앱 제출**: 팬이 선수 선택 → 응원문구/이미지 업로드 → 동의 체크.
- **AI 필터+요약**: 비방/혐오/개인정보 자동 필터 → 중복·스팸 합치기 → 다국어 번역.
- **대표계정 게시**: 일정 주기마다 “선수별 응원 하이라이트”를 한 장(혹은 캐러셀/스레드) 로 올리고, 가능하면 `@멘션/태그`
- **투명성**: 원출처(닉네임/국가/언어) 최소 표시, 비공개 요청 시 익명 처리
- **선수 UX**: 댓글 바다 속에서 건지는 게 아니라, 대표계정 타임라인*만 보면 응원 파악 끝

### 핵심 기능 설계

#### 1) 제출 & 선수 선택

- 선수 검색(국가/종목/이름/핸들), 즐겨찾기, 최근 트렌드(요청량/경기일정 기반).
- 폼: 텍스트(최대 280~500자), 이미지 1–3장, 언어 자동감지.
- 동의: 이용약관(재게시 허용/2차가공 동의), 개인정보 최소수집.

#### 2) AI 콘텐츠 파이프라인

- **Moderation**: 욕설/혐오/개인정보(전화/주민번호 등), 광고 링크 컷.
    - OpenAI Moderation(or Perspective API) + 정규식 + 금칙어 사전.
- **클러스터링/요약**: 같은 의미 묶어서 **대표 문장** 생성(다국어→선수 언어/영어 번역).
- **이미지 합성**:
    - A안: 한 장 포스터(불꽃/국기/선수명 + 대표 응원문 3–6개 + 기여국가 배지).
    - B안: 인스타 캐러셀(1장=대표, 2–4장=국가/언어별 하이라이트).
    - C안: X(트위터) 스레드: 대표 트윗 + 답글로 국가/언어별 1줄 요약.
- **TTS(선택)**: 스토리/릴스용 15–30초 낭독 클립 생성.

#### 3) 게시 & 스케줄링

- **시간대**: 선수/대회 로컬(KST ↔ CET) 교차 포스팅.
- **빈도 관리**: 선수별/플랫폼별 쿨다운(예: @멘션 하루 1–2회), 스팸 회피.
- **형식 매핑**:
    - **Instagram/Threads**: 캐러셀+자막, `@멘션`(정책 제약 하단 참고), 해시태그(국가/종목/이벤트).
    - **X(트위터)**: 대표트윗 + 스레드, @멘션, 링크는 최소화.
    - **YouTube Shorts**(옵션): 20초 음성요약+자막.

#### 4) 선수/에이전시용 뷰

- 선수 전용 링크(혹은 DM): 최근 24–48시간 하이라이트만 모아보기(웹뷰).
- 구단/연맹/에이전시에 합법적 재활용 허용(브랜딩 가이드 제공).

## 생각하고 보완할 점

### 1. SNS 플랫폼 정책 및 제약

#### 인스타그램 / 쓰레드

- 사람이 운영하는 계정에서 `@멘션`을 달면 알림이 감
- 하지만 서비스 대표 계정 같은 비즈니스 계정은 안 갈 확률 높음. 자동 포스팅이기 때문.

#### X(트위터)

- 트윗 내 `@멘션`은 계정 알림 탭에 표시됨.
- 단, 대상이 내 계정을 차단/뮤트/팔로우X, Verified 계정이면 필터링될 가능성이 크다는 것
- 짧은 시간 여러 멘션 반복해도 트윗은 표시되지만 알림 비활성화

#### 틱톡, 유튜브, 페이스북

- 태그 기능 있지만, 영상/설명란 내 태그는 알림 제한이 있음
- 댓글 멘션은 대부분 알림 감
- 숏츠의 경우, `@username`이 자동 링크로 연결되지만, 유튜버가 멘션 알림 끄면 무시됨

#### 해결 방안

1. 대표계정 자동포스팅. 하지만 태그 안하고 노출용으로만...
2. 관리자가 하루 1회 대표 계정의 게시물에 해당 선수의 태그 직접 남기기
3. 선수 전용 페이지같이 개인 링크로 응원 하이라이트 바로 확인 가능하게 하기
