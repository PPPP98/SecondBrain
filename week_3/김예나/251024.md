# 2025.10.24

## CSR(client side rendering)

- 빈 html을 받고 브라우저(클라이언트)에서 js가 만듦
- 장점 : 초기 로딩 후 페이지 간 이동이 빠름
- 단점 : 초기 로딩 속도가 느리고 SEO에 불리

## SSR(server side rendering)

- 사용자가 페이지를 요청할 떄마다 서버에서 새로 만듦
- 장점 : 항상 최신 데이터를 보여줌, SEO에 유리
- 단점 : 매번 서버가 동작하기 때문에 서버 부하 有, 느림

## SSG(static site generator)

- 사용자에게 서비스하기 전에
- 빌드 시점에 필요한 모든 페이지를 완전한 HTML로 미리 생성해둠
- 장점 : 속도가 가장 빠름, 보안성 ↑, 서버 비용 저렴
- 단점 : 데이터가 자줒 바뀌면 매번 다시 빌드 必 (블로그, 기술 문서에 적합)
